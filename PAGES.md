# صفحات موقع «أيتامنا» — دليل التصفّح

هذا الملف يسرد كل صفحات الموقع ومساراتها، ليتمكّن أي مبرمج فرونت من تصفّح الموقع بسرعة.

## كيف تشغّل الموقع وتتصفّحه

```bash
npm install      # أول مرة فقط
npm run dev      # يشغّل خادم التطوير (Vite)
```

بعد التشغيل سيظهر رابط محلي مثل: `http://localhost:5173/`
افتح أي صفحة بإضافة اسم الملف بعد الرابط، مثال:
- الرئيسية: `http://localhost:5173/index.html`
- تسجيل الدخول: `http://localhost:5173/login.html`

> أو افتح النسخة المبنية: `npm run build` ثم `npm run preview`.

## 3) مسارات التنقّل المترابطة (Flow)

**المصادقة:**
```
login → (نسيت كلمة المرور؟) → forgot-password → reset-verify → reset-password
login → (إنشاء حساب) → register → verify → verify-success
```

**التنقّل العام (من الهيدر/الفوتر، مفعّل في كل الصفحات):**
- تسجيل دخول → `login.html`
- تسجيل جمعية → `register.html`
- المبادرات → `initiatives.html`
- الجمعيات → `associations.html`
- الدورات → `courses.html`
- تواصل معنا → `contactus.html`
- الأسئلة الشائعة → `faq.html`
- الشروط والأحكام → `terms-conditions.html`
- سياسة الخصوصية → `privacy.html`
- أيتامنا مطمئنون → `reassured.html`

## 4) ملاحظات تقنية سريعة

- **نوع الهيدر:** الصفحات العامة تستخدم هيدر ثابت `fixed-header` بتأثير عند التمرير؛ صفحات «التفاصيل» (blog-details, event-details, initiatives-details, news-details, research-details) تستخدم هيدر `sticky`.
- **الموبايل:** أيقونة برجر تفتح قائمة منزلقة (drawer) من اليسار في كل الصفحات العامة.
- **CSS/JS:** كل التنسيق في `src/style.css` وكل التفاعل في `src/main.js`.
- **متجاوب 100%:** موبايل / تابلت / ديسكتوب.

> ملف منفصل لخطة التحويل إلى Laravel موجود في `LARAVEL-INTEGRATION.md`.
